OPTION 2: MANUAL STEP-BY-STEP
------------------------------

Step 1: Start Infrastructure (5 min)
    cd monitoring
    docker-compose up -d
    cd ..

Step 2: Verify Services Running
    docker-compose -f monitoring/docker-compose.yml ps
    (All should show "Up")

Step 3: Setup Database (5 min)
    cd database
    python generate_data.py
    
    $env:PGPASSWORD = "postgres"
    psql -U postgres -h localhost -c "CREATE DATABASE benchmark;"
    psql -U postgres -h localhost -d benchmark -f schema.sql
    psql -U postgres -h localhost -d benchmark -f categories.sql
    psql -U postgres -h localhost -d benchmark -f items.sql
    cd ..

Step 4: Start a Variant (choose one)
    .\RUN_VARIANT.ps1 A    # JAX-RS on port 8081
    .\RUN_VARIANT.ps1 C    # Spring MVC on port 8082
    .\RUN_VARIANT.ps1 D    # Spring Data REST on port 8083

Step 5: Test It
    curl http://localhost:8082/actuator/health
    curl http://localhost:8082/categories?page=0&size=10
    curl http://localhost:8082/items?page=0&size=10

================================================================================
  NEED HELP?
================================================================================

Issue: Build fails
Solution: .\mvnw.cmd clean install -DskipTests -U

Issue: Port already in use
Solution: netstat -ano | findstr :8082
         taskkill /PID <pid> /F

Issue: Database connection refused
Solution: docker-compose -f monitoring/docker-compose.yml restart postgres

Issue: Out of memory
Solution: java -Xms2g -Xmx4g -jar <variant>.jar

Full troubleshooting guide in EXECUTION_GUIDE.md
